10.1 Syntax

The easiest way to translate a string or variable is to enclose it between {_ and _}: 

{_ var _}
{_ "hello" _}

Next: Filters, Previous: Syntax, Up: Internationalization   [Contents][Index]

10.2 Tags

  • trans

Up: Tags   [Contents][Index]

10.2.1 trans

Translates a variable or string

Example: 

{% trans var %}
{% trans "hello" %}

10.3 Filters

  • trans

Up: Filters   [Contents][Index]

10.3.1 trans

Translates a variable or string.

For example: 

{{ var | trans }}
{{ "my string" |  trans }}

Next: Backends, Previous: Filters, Up: Internationalization   [Contents][Index]

10.4 Choosing language

To choose the language to use, set the *CURRENT-LANGUAGE* variable.

For example: 

(let ((djula:*current-language* :es))
   (djula:render-template* +translation.html+))

Previous: Choosing language, Up: Internationalization   [Contents][Index]

10.5 Backends

Djula supports two backends for doing translations: cl-locale1 and gettext2

Please have a look at the demo and the documentation of those packages to figure out how to use them.
